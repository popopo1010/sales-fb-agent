# CAヒアリングFBシステムの課題感・マニュアルの不足・論点

CAの初回面談ヒアリングをFB（フィードバック）するシステム（Slack /fb）と、現行マニュアル（19-onboarding-prex等）の関係を整理し、足りないものと論点を明示する。

---

## 1. FBシステムの前提整理

### 1.1 システムの流れ

```
CAが初回面談実施
    ↓
書き起こしをSlack /fb で投入
    ↓
FB生成（PSS + OPS + fb_format を参照）
    ↓
#dk_ca_初回面談fb に投稿（担当メンション＋候補者名付き）
```

### 1.2 FB生成が参照するドキュメント

| 参照先 | 内容 |
|--------|------|
| **reference/pss** | PSS営業スキーム |
| **reference/operations** | オペレーションマニュアル（候補者心理・ヒアリングフロー・6項目等） |
| **config/fb_format.md** | FB出力形式（8項目の定義） |

**重要**: **reference/domain/construction（19-onboarding-prex等）はFB生成に参照されていない。**

---

## 2. マニュアルで足りないもの（ギャップ分析）

### 2.1 FBシステムが参照していないドメイン知識

| 所在 | 内容 | FB生成での利用 |
|------|------|----------------|
| 19-onboarding-prex | 電気職種パターン・年収（再エネ550〜650万、工場600〜700万等） | **未参照** |
| 19-onboarding-prex | 障壁別ヒアリングの具体的な言い回し | **未参照** |
| 19-onboarding-prex | 施工管理の前提（ゼネコン系NG、50〜100名規模等） | **未参照** |
| 19-onboarding-prex | おやすみの感覚（年間休日100日＝建設業に多い等） | **未参照** |
| 04-recruitment-playbook | セグメント別訴求・候補者タイプ×セグメント | **未参照** |
| 13-qualifications | 資格の数値・訴求 | **未参照** |

→ **FB生成のプロンプトにドメイン知識が入っていないため、建設・電気領域に特化したFBが出にくい。**

### 2.2 fb_format.md の年収レンジ vs 19-onboarding-prex の年収

| 項目 | fb_format.md（FB生成で使用） | 19-onboarding-prex（FB未参照） |
|------|------------------------------|-------------------------------|
| 電気工事士 | 第2種・経験なし360〜420万、第1種450〜600万 | 電気工事士400〜450万（標準）、500〜600万（残業多め） |
| 施工管理 | 施工管理経験あり600万以上 | 未経験400〜550万（〜600万）、経験者600〜800万 |
| 職種パターン | なし | 再エネ550〜650万、工場600〜700万、ビルメン400〜450万 |

→ **fb_formatの年収レンジが粗く、職種パターン（再エネ・工場・ビルメン等）の切り分けがない。**

### 2.3 障壁の扱いのズレ → **一部解消（論点3実装済）**

| 観点 | FB出力の「障壁」 | 障壁突破の質問 |
|------|------------------|----------------|
| 目的 | 商談を進める上で注意すべき点（FBの**出力**） | 障壁に対してCAが**どう聞くか**の言い回し |
| 実装 | 6. 障壁で指摘 | 8. 残論点でOPS barrier-interview-questions.md を参照し、具体的な質問フレーズを出力 |

→ **論点3実装により、FBの残論点に障壁タイプ別の具体的な質問フレーズが含まれるようになった。19-onboarding-prexの障壁別ヒアリングはCA向けオンボ用（建設特化の言い回し含む）。**

### 2.4 ヒアリングの深掘りレベル評価との接続

fb_formatでは「OPS 3-2bis」を参照して5観点で採点するが、19-onboarding-prexには以下がない。

- **深掘りレベル（Lv1/Lv2/Lv3）の判定基準**の簡易版
- **6項目の優先順位**が曖昧なときの「2択で聞き直す」具体例（19には障壁別はあるが、優先順位用の2択例はOPSに依存）
- **電気・施工管理特有**の深掘り観点（例：実務経験の対象工事か、高圧経験の有無等）

→ **OPSは汎用、19は建設ドメイン特化。FBはOPSベースで、建設特化の深掘り観点がプロンプトにない。**

### 2.5 書き起こしの質・投入方法

| 課題 | 現状 | マニュアルでの記載 |
|------|------|-------------------|
| 書き起こしの粒度 | 音声→文字起こしの精度・区切り方にばらつき | 19に「書き起こしの取り方」のガイドなし |
| 必須ヒアリング項目の抜け | エリア・資格・年収・6項目等が聞けていないとFBの質が落ちる | OPSにフローはあるが、19には「FB投入前に確認すべき項目」がない |
| 面談時間 | 「2〜3分」と伝えて30〜40分かかっても問題ない（OPS） | 19には記載なし。FBの改善点には含めないルールあり |

→ **CAが「何を聞いておくべきか」はOPSにあるが、「建設領域で特に聞くべきこと」は19に分散。FB投入前チェックリストはない。**

## 3. 論点（検討すべきテーマ）

### 論点1: FB生成にドメイン知識を載せるか → **ドメインなしでOK（決定）**

- **決定**: プロンプト肥大化を避けるため、reference/domain/construction はFB生成に追加しない。
- **現状**: PSS + OPS のみ参照。建設・電気の専門知識はプロンプトに含まれない。

### 論点2: 年収レンジの単一ソース化

- **現状**: fb_format.md と 19-onboarding-prex で年収レンジが別定義。整合が取れていない。
- **選択肢**:
  - A) fb_formatの年収表を19の内容で更新し、職種パターン（再エネ・工場・ビルメン等）を追加
  - B) 19を正とし、fb_formatは19を参照する形に変更（自動読み込み or 手動同期）
  - C) 年収はOPSのsalary-market等に集約し、fb_format・19の両方から参照
- **論点**: 更新頻度・責任の所在。年収は市場変動で変わるため、どこを正とするか。

### 論点3: 障壁の「指摘」と「突破の質問」の一体化 → **具体的な質問を入れる（実装済）**

- **実装**: OPS に barrier-interview-questions.md を追加。fb_format の「8. 残論点」で、6. 障壁で指摘した障壁タイプに対応する具体的な質問フレーズを必ず含めるよう指示を追加。
- **参照**: reference/operations/barrier-interview-questions.md（FB生成時にOPS配下として自動読み込み）

### 論点4: FB投入前チェックリストの整備

- **現状**: 「書き起こしを貼り付けて送信」のみ。必須項目の抜けを防ぐ仕組みなし。
- **選択肢**:
  - A) /fb モーダルに「エリア・資格・年収・6項目の優先順位」等の簡易チェックを追加（未入力なら警告）
  - B) マニュアル（19）に「FB投入前に確認すべき項目」を明記。運用で担保
  - C) FB生成後に「要再ヒアリング」フラグを出し、不足項目をFB内で指摘（現状に近い）
- **論点**: 入力負荷の増加 vs 書き起こしの質担保。モーダルを重くしたくない場合、マニュアルでの運用が現実的。

### 論点5: 建設領域特化の深掘り観点の定義

- **現状**: OPSの深掘りレベル（3-2bis）は汎用。電気工事士の実務経験、施工管理の現場規模等の観点はOPSに薄い。
- **選択肢**:
  - A) 19に「建設・電気領域のFB評価で見るべき観点」を追加し、FBプロンプトに含める
  - B) OPSに建設特化の補足セクションを追加
  - C) fb_formatの採点基準に「電気・施工管理の専門観点」を1観点追加
- **論点**: 汎用OPSとドメイン特化のバランス。他業界に展開するならOPSは汎用のまま、ドメインは参照として追加する設計がよい。

### 論点6: マニュアルの役割分担の明確化

- **現状**:
  - OPS: 候補者心理・ヒアリングフロー・FB評価基準（FB生成に使用）
  - 19-onboarding-prex: 建設ドメインの実務・障壁別言い回し（FB生成に未使用）
  - fb_format: FB出力形式（FB生成に使用）
- **論点**: 19を「CAが読むだけ」にするか、「FB生成にも使う」か。使うならgeneratorの変更が必要。使わないなら、19とFBの整合性をどう保つか（年収・障壁の言い回し等）。

## 4. アクション案（優先度イメージ）

| 優先度 | アクション | 効果 | 状態 |
|--------|------------|------|------|
| 高 | 論点2: 年収レンジの整合（fb_format ↔ 19） | FBの年収相場評価の精度向上 | 未着手 |
| - | 論点1: ドメイン追加 | - | **見送り**（プロンプト肥大化回避） |
| 中 | 論点3: 残論点に障壁別の具体的な質問フレーズを生成 | CAの次のアクションが具体化 | **完了** |
| 中 | 論点4: 19に「FB投入前チェックリスト」を追加 | 書き起こしの質・必須項目の担保 | 未着手 |
| 低 | 論点5: 建設特化の深掘り観点を定義 | FB採点の専門性向上 | 未着手 |
| 低 | 論点6: マニュアル役割分担の文書化 | メンテナンス方針の明確化 | 未着手 |
