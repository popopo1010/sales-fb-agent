# PSS（Professional Selling Skills）営業スキーム — 要約版

※ 原典: アチーブグローバル社 PSSプログラムテキストを評価基準として要約

---

## 基本理念

- **ニーズ**: 何かを改善、あるいは達成したいというお客さまの要望
- お客さまのニーズを理解し満たすことが、お客さまの成功と自社の成功の両方につながる
- 面談の目的: お互いの利益となる決定に至るよう、ニーズに焦点を当てて十分に情報交換を行う

## 面談の4つの核となるスキル

1. **オープニング** — 話し合う内容についての情報交換
2. **プロービング** — ニーズに関する情報収集
3. **サポーティング** — ニーズに関する情報提供
4. **クロージング** — 協力して進むべき次のステップについての情報交換

通常、複数のニーズを考慮して意思決定するため、プロービングとサポーティングは面談中に繰り返し行う。

---

## ユニット1: オープニング

### 目標
話し合う内容について合意に達すること

### 使うタイミング
お互いに本題に入る用意ができているとき

### 手順（4ステップ）
1. **面談の位置づけ** — お客さまが認識していることと今回の面談の関連を述べる
   - 面識あり → 前回の内容を振り返り、つながりを持たせる
   - 初対面 → 訪問の経緯やお客さまの関心事を述べる
2. **話題の提案** — 面談で話し合いたい内容を伝える（会話の方向を定める）
3. **価値の提示** — その話題がお客さまにとってどんな意味を持つか説明する（※商品の価値ではなく話題の価値）
4. **同意のチェック** — 提案した話題にお客さまが同意したか確認する

### 事前準備
- お客さまは何のためにセールスと面談するのか？
- 自分は何のためにこのお客さまと面談するのか？
- 営業プロセスの次のステップへ進むために話し合うべきことは何か？

---

## ユニット2: プロービング

### 目標
ニーズについて**明確**、**完全**に、そして**認識のずれなく**理解すること

- **明確に理解する**: ニーズ一つひとつについて、具体的に何を達成・改善したいか、なぜそのニーズが重要か、なぜそのニーズを持ったか
- **完全に理解する**: すべてのニーズと、それらの優先順位
- **認識のずれなく理解する**: お客さまがイメージしているニーズの絵とまったく同じ絵をセールスが描ける

### 使うタイミング
お客さまから情報を引き出したいとき

### 何を探るか

#### お客さまの状況
- お客さまを取り巻く環境の事実・状態・できごと、それに対する気持ちや意見
- 状況を探ることで、なぜお客さまがニーズを持ったのか理解できる
- 企業の場合、状況は層構造: 業種/業界 → 会社全体 → 部門 → 個人

#### お客さまのニーズ
- **ニーズを表す言葉**: 「〜したい」「〜を望む」「〜が必要だ」「目的は〜」「〜を探している」「重要なのは〜」「〜に関心がある」「〜するのが目標だ」「解決したいのは〜」「〜しなければならない」
- お客さまの話にしっかり耳を傾け、ニーズを表す言葉を聞き分けることが重要

#### 具体的に何か
- お客さまのニーズの具体性は千差万別。はっきり理解できるまでプロービングを続ける

#### ニーズの裏のニーズ
- お客さまが達成したいと思っている、より大きな目標
- ニーズの裏のニーズを探ることで、ニーズがなぜ重要かが理解できる
- 企業の場合、大きく3つの業務上の分野: **財務** / **生産性** / **イメージ**

### プロービングの種類

#### 拡大型プロービング（オープンクエスチョン）
- お客さまに自由に答えてもらう
- 例:「どのようにしたいとお考えですか？」「なぜそうされているのですか？」
- 用途: 状況やニーズの情報収集、仮説をもとにお客さまの考えを聞く

#### 限定型プロービング（クローズドクエスチョン）
- 「はい/いいえ」・選択肢・数量で答えてもらう
- 例:「〜で間違いありませんか？」「最も重要なものはどれですか？」
- 用途: 特定の情報取得、理解の確認、ニーズ存在の確認

#### 効果的なプロービングのポイント
- **簡潔に行う** — 要点をまとめて
- **一つひとつ行う** — 一度に多くのプロービングをしない
- **誘導しない** — お客さまに自由に話してもらう

### プロービングの姿勢
- **お客さまのビジネス全体を把握し、お客さまの課題をともに深く考える**
- ビジネス全体を把握して仮説を立て、より深いレベルで話し合う
- より深く掘り下げたプロービング、視点を切り替え違った角度からのプロービング

## ユニット3: サポーティング

### 目標
ニーズを満たすために何ができるか、その具体的な方法をお客さまに理解してもらうこと

### 使うタイミング（3条件すべて）
1. お客さまがニーズを口にしたとき
2. お互いにニーズを明確に理解しているとき
3. 商品や会社がどのようにニーズを満たすか、セールスがわかっているとき

※ 条件を満たす前にサポーティングすると: 押し売りと感じられる、情報が正しく伝わらない、最適でない商品を紹介するリスク

### 手順（3ステップ）

#### 1. ニーズの受け止め
- お客さまのニーズを理解し尊重していることを示す
- 方法:
  - ニーズは重要で価値があることに同意する
  - ほかの人や企業にとっても関心が高いことを伝える
  - ニーズに関するお客さまの気持ちに気づいていることを伝える
- 画一的でなく、場に応じてさまざまな受け止めの言葉を使う

#### 2. 関連する特徴と利点の紹介
- **特徴**: 商品や会社の持つ特性（事実）
- **利点**: 特徴が持つお客さまにとっての価値（ニーズをどう満たすか）
- 紹介する特徴と利点は、サポーティングしようとする**特定のニーズに関連のあるもの**だけ
- 特徴と利点はどちらが先でもよい
- **ニーズの裏のニーズと利点の結び付け**: より大きな目標にまで利点を結び付けると効果的

#### 3. 同意のチェック
- 説明が理解され、利点が受け入れられたことを確認する
- 同意してくれた利点はメモしておく（クロージングで振り返るため）
- 利点を理解していない/同意していない場合はプロービングで明らかにする

## ユニット4: クロージング

### 目標
適切な次のステップについて合意に達すること（契約だけでなく、企画書提出・プレゼン実施・見積提出など含む）

### 使うタイミング
- お客さまが商談を前に進めてもよいというシグナルを出しているとき
- あるいは、お客さまが紹介した利点に同意してくれたとき

### 手順（3ステップ）
1. **利点の振り返り** — お客さまが同意してくれた利点をまとめて振り返る
2. **お互いの実施内容の提案** — お客さまとセールス双方が次に何をするか具体的に提案（お客さまだけの負担にならないようバランスをとる）
3. **同意のチェック** — 次のステップにお互い同じ思いで進もうとしているか確認

### お客さまが同意しない場合
- **ためらう場合**: プロービングで理由を明らかにし、心配があれば解消する。それでもためらうなら提案内容を修正し、結論の期限だけは押さえる
- **はっきり断られた場合**: お礼を述べ、可能ならコメント（決め手は何か、不足は何か）をもらい、将来の関係維持の許可を得る

## 補足: 会話を円滑にするスキル

面談のどの場面でも活用できる「聞き手」としてのスキル:

1. **受け止め** — ニーズの受け止め / 一般的な言葉の受け止め / 心配ごとの受け止め
2. **確認** — 「つまり〜ということですよね？」と聞いた内容を要約して確認
3. **位置づけ** — 面談の位置づけ、探る意図の説明、振り返りの導入、関連づけ
4. **同意のチェック** — 各段階での合意確認

## ユニット5: 無関心の克服

### お客さまが無関心な理由
- 競合会社の商品を使用し満足している
- 現状を改善できることに気づいていない
- 現状を改善することの重要性に気づいていない

### 手順（3ステップ）

#### 1. お客さまの見解の尊重
- 現状への満足を理解し尊重する姿勢を示す（押し売りでないことを伝える）

#### 2. 話を聞くことの許可依頼
- オープニングスキル（話題の提案・価値の提示・同意のチェック）を使い、限られた話題についてたずねてもよいか許可をもらう
- 話の広がりと時間の両方を限定する

#### 3. ニーズに気づいてもらうためのプロービング
以下の流れで実施:
1. **(1) お客さまの状況を探る** — 無関心なお客さまはニーズを持っていないと思い込んでいるため、まず状況から探る
2. **(2) オポチュニティを特定する** — 改善の手助けができる可能性（問題や状態）があるかどうかを特定する
   - **オポチュニティ**: セールスの商品や会社の特徴や利点によって、改善の手助けができる可能性（問題や状態）
   - オポチュニティ ≠ ニーズ（お客さまが「改善したい」と思わなければニーズにならない）
3. **(3) 影響を明らかにする** — 現状を放置したときどのような影響が及ぶかを明らかにする（現在・将来含む）
4. **(4) ニーズの存在を確認する** — 「ニーズを表す言葉」を盛り込んだ限定型プロービングでニーズの有無を確認

### 事前準備
- お客さまの見解を尊重し許可を得る言葉を考える
- オポチュニティを予想する（特徴と利点表を活用）
- 状況からオポチュニティを特定するプロービング（できるだけ拡大型）を考える
- オポチュニティがあった場合の影響を考え、影響を明らかにするプロービングを考える

## ユニット6: お客さまの心配ごとの解消

心配ごとは、お客さまがニーズを表現する方法の1つ。面談のどの段階でも起こる。

### 心配ごとの3タイプ

| タイプ | 定義 | ニーズとの関係 |
|--------|------|----------------|
| **不審** | 紹介した特徴や利点をお客さまが疑っている | 満たせるニーズがある |
| **誤解** | 提供できる特徴や利点を、提供してもらえないとお客さまが誤って考えている | 満たせるニーズがある（背景に隠れている） |
| **不満** | ある特徴や利点に対してお客さまが満足していない | 満たせないニーズがある |

### 共通の最初のステップ
**心配ごとを理解するためのプロービングの実施**
- 心配ごとが何かについての理解が十分でないときは、確実にわかるまでプロービングを行う
- タイプがわかっている場合でも、背景にあるニーズを十分に理解するために最初にプロービングを行う

### 不審の解消手順
1. **心配ごとの受け止め** — 理解し尊重している姿勢を示す（ただし「まったくそのとおり」等、いっそう心配になる言い方はしない）
2. **関連する証拠の提示** — 特徴や利点が備わっていることを裏付ける証拠を提示（データ・サンプル・デモ・記事・感謝状・信用照会状・契約書・仕様書・テスト結果等）
3. **同意のチェック** — 証拠を受け入れ同意したか確認

### 誤解の解消手順
1. **心配ごとをニーズに置き換え確認** — 後ろ向きな言葉を「〜したい」という前向きなニーズの表現に置き換えて確認する（限定型プロービング使用）
2. **サポーティングの実施** — ニーズの受け止め → 関連する特徴と利点の紹介 → 同意のチェック

### 不満の解消手順
1. **心配ごとの受け止め** — 理解し尊重している姿勢を示す
2. **より大きな視点に注目** — 全体像から考えてもらえるよう促す（※「しかし」は使わない。「そこで〜」「〜も事実ですね」等を使う）
3. **受け入れられた利点の重要性を強調** — すでに受け入れられた利点を振り返り重要性を強調する
   - 選択の観点: お客さまの最重要ニーズを満たす利点 / ニーズの裏のニーズを満たす利点 / 競合には提供できない利点
4. **同意のチェック** — 満たせるニーズにより最終的な結果を得るのに役立つと思ってもらえたか確認

## 評価で着目すべきPSSスキル一覧

| スキル | 核となるポイント |
|--------|-----------------|
| オープニング | 面談の位置づけ → 話題の提案 → 価値の提示 → 同意のチェック |
| プロービング | 拡大型/限定型の使い分け、ニーズの裏のニーズの深掘り、明確・完全・認識ずれなし |
| サポーティング | ニーズの受け止め → 関連する特徴と利点の紹介 → 同意のチェック |
| クロージング | 利点の振り返り → お互いの実施内容の提案 → 同意のチェック |
| 無関心の克服 | 見解の尊重 → 許可依頼 → 状況→オポチュニティ→影響→ニーズ確認 |
| 心配ごとの解消 | 不審=証拠提示 / 誤解=ニーズに置き換え+サポーティング / 不満=大きな視点+受入利点強調 |
