## ― 候補者の意思決定フレームワーク × 実務オペレーション ―

---

# 第1章｜全体設計思想

## 1-1. このマニュアルの構造

本マニュアルは「候補者の意思決定プロセス」を軸に、CAの全オペレーションを再編成したものである。

従来のCAオペレーションは「架電→ヒアリング→求人紹介→面接→内定→入社」という**自社業務フロー**を基準に設計されていた。しかし、本来CAが向き合うべきは「候補者の頭の中で何が起きているか」であり、**候補者の意思決定プロセス**を軸にオペレーションを再設計することで、各アクションの意味と効果が明確になる。

```
【従来の設計】自社の業務フロー基準
  架電 → ヒアリング → 求人紹介 → 面接 → 内定 → 入社

【本マニュアルの設計】候補者の意思決定プロセス基準
  動機形成 → 探索・接触 → 評価・比較 → 意思決定 → 実行・定着
    ↑ 各フェーズに対応するCAのアクションを配置
```

## 1-2. CAのマインドセット（大原則）

### 原則①：御用聞き禁止 →「条件の背景を聞く」が正義

条件は**ニーズの"手段"であり、ニーズそのものではない。**

候補者が「年収500万以上がいい」と言った時、その背景には「子供の教育費を確保したい」「住宅ローンを安心して払いたい」「自分の仕事が正当に評価されている実感がほしい」といった本質的なニーズがある。

CAの価値は「条件を聞いてマッチングする」ことではなく、**「背景（Why）→ 優先順位 → 現実の落とし所 → 次の一手」を設計する**こと。

### 原則②：否定しない（受容）→ 受け止めはセンスではなく手順

候補者の発言を否定した瞬間、心理的な壁ができる。求職者の欲求そのものを変えることはできないと思った方がいい。だからこそ別の欲求を見つけて受け入れるコミュニケーションを取る。

**受け止めの5ステップ：**

| Step | フレーズ例 |
| --- | --- |
| ①共感/尊重 | 「そう思われますよね」 |
| ②理由の肯定 | 「その条件を大事にしたい背景があるからですよね」「他の方も同じ点を気にされるのでとてもよくわかります」 |
| ③確認 | 「認識ずれないように確認させていただきたいのですが、今の発言の意図は○○の理解で合ってますか？」 |
| ④許可 | 「その背景まで分かると、より〇〇さんに最適な提案ができるので少しだけ聞いてもいいですか？」 |
| ⑤深掘り | 本質的なニーズへの質問 |

**簡易版：**「そうですよね、その前提わかると提案がズレないので、1個だけ聞いてもいいですか？」

**TIP（心理学）｜心理的リアクタンスの回避**
否定や押し付けは、人間が持つ「選択の自由を守りたい」という欲求（心理的リアクタンス）を刺激し、拒否や反発を招く。許可を取ってから深掘りする5ステップは、候補者の自律性を尊重し、心理的リアクタンスを低減する。

### 原則③：「決めさせる」のではなく「決められる状態をつくる」

一流のCAは候補者を操作しない。候補者が合理的に判断できるための**材料と構造**を提供し、結果として候補者が「自分で決めた」と感じる状態をつくる。

**TIP（心理学）｜自己決定理論（内的動機づけ）**
人は「自分で選んだ」と感じた行動ほど持続し、成果も出やすい（内的動機づけ）。外部から押し付けられた選択は短期では動くが、離脱や後悔につながりやすい。CAは「決めさせる」のではなく「決められる材料を渡す」ことで、候補者の内的動機づけを高める。

# 第2章｜意思決定の5フェーズモデル

候補者の転職意思決定は以下の5フェーズで進む。一直線ではなく、行きつ戻りつしながら進む非線形プロセスである。

```
Phase 1        Phase 2        Phase 3        Phase 4        Phase 5
動機形成   →   探索・接触   →   評価・比較   →   意思決定   →   実行・定着
(Trigger)      (Search)       (Evaluation)   (Decision)     (Action)

  ← 停滞・後退・離脱のリスクが各フェーズに存在 →
```

CAの仕事は、候補者が**今どのフェーズにいるか**を正確に把握し、**そのフェーズに適した支援**を行うこと。Phase 1の候補者に求人を送りつけても響かないし、Phase 4の候補者に業界説明をしても意味がない。

**TIP（心理学）｜変化のステージモデル（プロチャスカ）**
人は行動変容に「前熟考期→熟考期→準備期→実行期→維持期」の段階を踏む。フェーズに合わない介入（例：熟考期の人に「今日決めて」）は無効か逆効果。CAは候補者の「現在地」を把握し、その段階に合わせた介入の強さ・種類を選ぶ。

# 第3章｜Phase 1：動機形成（Trigger）

## 3-1. このフェーズの候補者心理

「なんとなく転職を考え始めた」から「行動に移す」までの段階。候補者の内側で転職への動機が芽生え、エネルギーが蓄積される。**ここで形成された動機の質と強度が、最終的な意思決定の成否を決定する。**

| トリガーの種類 | 具体例 |
| --- | --- |
| **Push要因**（現状からの逃避） | 人間関係の悪化、給与への不満、長時間労働、将来性への不安、ハラスメント |
| **Pull要因**（理想への接近） | スキルアップ欲求、年収アップ、新しい挑戦、業界への興味 |
| **ライフイベント** | 結婚・出産、親の介護、引越し、子供の進学、住宅購入 |
| **外部刺激** | 知人の転職成功、SNS・広告での求人接触、スカウト受信 |

**構造的ポイント：** Push要因だけで動いている候補者は「逃げの転職」になりやすく、意思決定の軸がブレやすい。Pull要因と組み合わさることで意思決定が安定する。CAは、Push要因で動き始めた候補者に対して、**Pull要因を言語化させる**ことで動機を構造化する。

**TIP（心理学）｜接近動機と回避動機**
心理学では「〜を得たい」（接近）と「〜から逃れたい」（回避）は別の動機システム。回避動機だけでは不安が支配し、意思決定が揺らぐ。CAは「何から逃げたいか」を聞いた上で「何に向かいたいか」を言語化させ、接近動機を活性化する。

## 3-2. CAのアクション：動機の3層構造化

候補者の転職理由を「表面」「中層」「深層」の3層で理解する。

```
表面：「給料を上げたい」
  ↓ なぜ？
中層：「子供の教育費がかかるから」
  ↓ それだけ？
深層：「父親として家族を養えている実感がほしい」
```

深層の動機が分かれば、提案の精度が劇的に上がる。上記の例なら、年収だけでなく「家族手当が充実」「転勤なしで家族と過ごせる」といった提案が刺さる。

**TIP（心理学）｜5つのWhy（原因の深掘り）**
「なぜ？」を繰り返すと、表面的な理由から本質的な欲求に到達する。深層の動機は本人も言語化できていない場合が多い。CAは「なぜその条件が欲しいのか？」を優しく繰り返し、本人が気づいていないニーズを引き出す。

**実際のヒアリングで使う質問：**

- 「今お話をお伺いしていると、xxxさんは電気工事士として現職でも活躍されているんじゃないかなと思ったんですが、今回求人を見ようと思われたきっかけはなんだったんですか？」
- 「現状の状態が続いた時に、一番いやなことってなんですか？」（→現状維持のコストを想起させる）
- 「それを解決したいですか？」（→YESまで引き出す）
- 「もし今の会社のここだけが変わったら、残りますか？」
- 「5年後、どうなっていたら"転職してよかった"と思えますか？」

## 3-3. 未経験者 vs 経験者：動機形成の違い

| 対象 | 典型的な動機 | CAの介入 |
| --- | --- | --- |
| **未経験者** | 業界チェンジ、キャリアリセット、将来性への投資、「手に職をつけたい」 | 業界の魅力と将来性を具体的に伝える。「こういう未来が手に入る」というビジョンを描く。Push要因だけでなくPull要因を言語化させる |
| **経験者** | 待遇改善、ステップアップ、環境改善、人間関係のリセット | 現状の不満を深掘りし根本原因を特定する。「転職で本当に解決するのか」を一緒に検証する |

# 第4章｜Phase 2：探索・接触（Search）

## 4-1. このフェーズの候補者心理

エージェント登録、求人サイト閲覧、知人への相談など、外部との接点を持ち始める段階。「まだ本気じゃない」「いい話があれば」という心理が支配的で、コミットメントが低い。レスポンスが遅い、面談をキャンセルする、といった行動が頻発する。

**このフェーズの候補者がやっていること：**

- 市場価値の確認（自分がどの程度の条件で転職できるか）
- 業界・企業の情報収集
- 複数エージェントへの登録・比較
- 求人票の閲覧と初期スクリーニング

## 4-2. リード管理オペレーション

### 新規リード対応

1. Slack（xchange-電気関連）の **#notify-leadgen** にてリードCV通知に対してスタンプを押して対応
2. 初回架電して繋がらない場合 → 「未通電」ステージへ

### 未通電対応（CA未接点な求職者・2週間管理）

このフェーズの求職者は**誰がかけてもOK**。最低2回は下記の時間でかける：

| 回数 | ルール |
| --- | --- |
| 1〜2回目 | 12:00〜13:00 または 18:00〜22:00 に架電 |
| 3回目以降 | 自由にかけてよい |

**留守電は残さない方がコールバック率が高い。**

### ナーチャリング

次の担当者が対応しやすいよう、ナーチャリング理由を選択してステージを変更する：

| 理由 | 備考 |
| --- | --- |
| 転職してしまった | ― |
| 関係が作れず対話終了 | ― |
| 情報を見たいだけ | **基本的に攻略可能**（Phase 2の候補者。下記「無関心の克服」を適用） |
| 期間内つながらず終了 | ― |

## 4-3. CAのアクション：初回架電

### オープニング（情報収集段階の候補者向け）

**Point：**

- 時間がかかる印象を与えないよう、質問は短く切って簡潔に伝える
- 「2〜3分」と伝えることで抵抗をなくすが、実際は30分程度話すことを目標にする
- 求職者の欲求を変えることはできないと思った方がいい。別の欲求を見つけるコミュニケーションを取る

**TIP（心理学）｜足場づけ（アンカリング）効果**
最初に提示された数字やフレームが、その後の判断に影響する。「2〜3分」と伝えることで、候補者は短い時間の枠で考えるため、電話を続ける心理的ハードルが下がる。反対に「30分お時間ください」から入ると拒否されやすい。

**スクリプト：**

> 「もしもし、xxさんでお間違いないでしょうか？」
「電気工事求人を検索いただいた件で連絡いたしました。今は情報収集をしたいということでお間違いなかったでしょうか？」
「電気領域専門エージェントの〇〇と申します。」
「xxさんの要望にあった求人を無料でご紹介させていただくために、ご登録内容の確認とご希望条件の確認に2〜3分程度お時間いただいてもよろしいでしょうか？」
> 

### ヒアリング（エリア → 資格/経験 → 希望条件 → キャリア方向性 → 転職きっかけ の順）

**① エリア**

> 「エリア、資格、希望条件の順番にお伺いさせていただきますね」
「先ず、お探しのエリアからお伺いさせて頂きたいのですが、ご登録いただいた〇〇県の〇〇市の周辺でお探しという形でお間違いございませんでしょうか。」
> 

※エリアが極端に狭い場合：「もし条件がよかったらどこまで可能か」を確認する。

**② 資格/経験**

> 「今お持ちの資格は、xxで間違いなかったでしょうか？具体的にはどのようなお仕事をされていますか？」
> 

※今の勤務先でその経験を積んでいるのか、転職して経験を重ねているかも把握する。

**③ 給与・待遇面**

> 「希望の年収と現在の年収はいかがでしょうか？」
> 

※ボーナスが出ていないケースや、残業代で積み増されているケースがあるので、**月の固定給と残業代は分けて聞く。**

**希望年収の理由を聞く時（＝動機の深層を掘る）：**

> 「仮に希望額のXXX万円を実現できたら、今の生活の中で**一番変えたいこと、叶えたいこと**は何ですか？（例：お子様の進学資金、住宅ローン、趣味、家族との時間への投資）」
「逆に、もしこのまま年収が上がらない状態が続くと、将来的に**どのようなリスクや不安**があるとお考えですか？」
> 

**最低金額が曖昧な場合/高すぎる場合：**

> 「最低限ここを割ると生活が成り立たない、というラインはどのあたりでしょうか？」
> 

**④ キャリアの方向性（未経験者向け）**

電気工事士のキャリアは下記の3つがある。どのキャリアがイメージに近いか聞く：

| キャリアパス | 年収レンジ | 特徴 |
| --- | --- | --- |
| **設備管理** | 400万程度〜 | 電気主任技術者へのアップデートが可能 |
| **電気工事士** | 400〜500万（〜600万） | 3〜5年で残業込み600万。希望レンジが高い人には600万を提示 |
| **施工管理** | 600〜800万（→1000万超） | 3〜5年で600〜800万。経験を積めば1000万超も可能 |

**未経験者向けキャリアパス詳細：**

**設備管理ルート：**

- 1年目：電気や機械の知識を付けて、メンテナンスの作業や点検作業、工具の使い方を学ぶ
- 3年目：一人で点検や報告書の作成が可能に。電気工事士等の資格取得タイミング。年収+50万円程度
- 5年目：後輩指導・大規模作業の管理・グループリーダーへ。電気主任技術者の資格取得タイミング。年収+80万円程度
- 7年目：部署のリーダー・役職付き、または専任の電気主任技術者として専門性を深めるかの選択肢

**施工管理ルート：**

- 1年目：さまざまな現場に先輩と同行、図面の読み方を学習
- 3年目：上司と二人で仕事がある程度でき、顧客折衝にも慣れる。図面から現場の想像がつくように
- 5年目：一人で施工管理者として現場を任される。電気工事施工管理技士の資格取得タイミング。年収+100万円程度
- 10年目：バリバリのプレイヤーとして継続するか、管理職へ移るかの選択肢。年収+200万円程度

**⑤ キャリアの方向付けの確認**

> 「今回の転職の後に、5年後、10年後にxxさんが『この道を選んでよかった』と振り返れるようにしたいので……」
> 
- 「今の仕事を続けていく中で、一番『将来が不安』と感じる部分はどこですか？（市場縮小？体力的な限界？）」
- 「もし条件が全て同じだとしたら、本当は**どんな現場・どんな案件**に携わっている時が一番ご自身らしいと感じますか？」

**ワークライフバランス vs 年収の比重確認：**

> 「極端かもしれませんが、年収600万で年間休日100日と、年収450万で休日120日以上ならどちらがイメージに近いですか？」
> 

※数値は相手に合わせて調整。

**TIP（心理学）｜認知的負荷の軽減**
選択肢が多すぎると人は決められない（選択のパラドックス）。「2択で選ばせる」ことで認知的負荷を下げ、優先順位を明確にできる。その上で「なぜそちらが近いですか？」と深掘りする。

以下で深掘る：

- 「『休みが欲しい』というのは、具体的に**どのような時間**を確保したいからでしょうか？（子供と遊ぶ、親の介護、副業、休息）」
- 「ご家族からは、現状の働き方について**具体的にどのような不満**（帰りが遅い、土日いない等）が出ていらっしゃいますか？」
- 「ご自身の感覚として、『これを超えると精神的・体力的に限界』と感じる残業時間や休日数のラインはありますか？」

**⑥ 転職のきっかけ（＝Phase 1の動機を再確認・深掘り）**

> 「今お話しをお伺いしているとxxxさんは、電気工事士として現職でも活躍されているんじゃないかなと思ったんですが、今回求人を見ようと思われたきっかけはなんだったんですか？」
> 

→ 転職の裏の理由を聞いていく。

> 「現状の状態が続いた時に、一番いやなことってなんですか？」
> 

→ 現状維持のコストを想起させる。

> 「それを解決したいですか？」
> 

→ YESまで引き出す。**ここで候補者自身がPush要因を自覚し、それを言語化することで動機が固まる。**

**⑦ ニーズの受け止め（確認）**

> 「齟齬ないように確認させていただければと思いますが、年収xxx万以上 / お休みxxx日以上 / ワークライフバランスor年収重視 / それを実現することでxxxのキャリアを築かれていきたいということでお間違いなかったでしょうか？」
> 
- *注意：転職のきっかけを聞かないと、あとで離脱率が大幅に増幅する。**ここで動機の構造化ができていないと、Phase 4で候補者が「やっぱりやめます」となるリスクが跳ね上がる。

### 逆オファーの提示（ニーズの確認がとれたら）

> 「xxさんのニーズにあった求人をご紹介させていただきますね。あわせて逆オファーを企業さまからいただくことも可能です。具体的には企業に『こんなご経歴のある方がいらっしゃって、年収xxx万以上で御社はどうでしょうか？』のような形で、年収を確約してオファーをもらってくることができます。年収が高く出やすいのですが、xxxさんのお役に立てそうでしょうか？」
> 

（OKが出たら）

> 「法人さんから逆オファーをいただくにあたり、履歴書では伝わらない部分を丁寧に説明させていただくことで、高い年収をいただくことができます。このまま基本情報・経験についてもお伺いできますか？」
> 

（YESの回答後）

> 「基本情報 ＋ 経歴に関する情報 ＋ 資格情報の3つになりますので、順番に聞いていきますね」
> 

→ 基本情報 → 経歴 → 資格情報の順にヒアリング

> 「ありがとうございました。こちらをもとに内容に齟齬ないかLINEでお送りさせていただきますので、後ほど追加お願いできますでしょうか？」
> 

### 信頼の醸成（逆オファーの同意がとれたら）

**ここが極めて重要。候補者に「このエージェントは押し売りしない」と感じさせるフェーズ。**

> 「いろいろお伝えしましたが、絶対に転職しなきゃいけないとかないですし、現職は現職で魅力的な部分があってこれまでやってこられていると思いますし、別に転職がすべてではないと思ってます。」
> 

> 「なので、あくまでも私もよく他の方にも話すんですけど、転職先が例えば内定が2つ出たとしても、どっちにしようかなよりは、**内定先の2つと、現職の3つの選択肢の中でどれが一番いいのか**を考えた方が良いと思っていて、その中で現職が一番いいと思えればそれはそれで納得感もって頑張れると思いますし、それはそれでいいのかなと思っています。」
> 

**このトークの本質：**
一流のコンサルは「短期の成約ではなく長期の信頼を優先する」。このスタンスを示すことで、候補者は「この人は自分の味方だ」と感じ、以降のプロセスでの離脱率が大幅に下がる。

**TIP（心理学）｜ソーシャル proof（社会的証明）の逆活用**
「転職しなくてもいい」と言うことで、CAは「売りたいだけの人」ではないと示す。人は「この人は自分の利益より私の利益を考えている」と感じると信頼する（利他的シグナル）。短期的な売上より候補者の利益を優先する姿勢が、長期的な成約とリファラルを生む。

### アポイント設定

| 候補者の温度感 | アポの取り方 |
| --- | --- |
| **転職意向が高い** | 「次回電話で条件に合う求人のご案内と、実際にxxさんに条件の中でも優先度付けをしていただければと思っておりますが、明日明後日で30分程度お時間作れるお時間いかがでしょうか？」（求人は事前送付） |
| **転職意向が低い** | 「本日お伺いした内容を整理いたしますので、xxさんのご希望の条件にお間違いがないかの確認と、合わせて求人情報をご用意させていただきます。明日明後日で30分程度で都合のいいお時間をいただけますでしょうか？」 |

**アポ設定のルール：**

- 基本**2日以内**に設定。1週間後以上を指定される場合は脈が薄い＝信頼されていない可能性が高い
- アポイントの**当日朝に必ずリマインドのLINE**を送る
- 相手の勤務時間や帰宅時間を想定して、都合の良さそうな時間を提案する

## 4-4. 「無関心の克服」テンプレート

「今はまだいい」「怪しい」という壁を突破するための型。

**見解の尊重 → 許可依頼 → オポチュニティ（機会）の特定** の流れで実施。

> ①尊重：「そうですよね。一生のことですから、慎重に動かれたいというお気持ち、よく分かります。」
> 
> 
> ②許可：「その上で、情報収集の段階だからこそ、『今、xx様のご経歴なら市場価値がいくらつくのか』という相場観だけでも知っておいて損はないかと思います。2〜3分だけ、現状の確認をさせていただいてもよろしいでしょうか？」
> 
> ③質問：「ちなみに、現職でこのままあと3年働いたとして、ご希望の年収〇〇万に届くイメージは湧いていますか？」
> 

→ 届かないなら、それが「今動くべき課題（オポチュニティ）」になる。

**TIP（心理学）｜損失回避と現状維持バイアス**
人は「得る喜び」より「失う痛み」を約2倍強く感じる（プロスペクト理論）。「このまま続くと届かない」と現状維持のコストを想起させることで、無関心の壁を崩しやすくなる。「市場価値を知る」という小さな得を提示しつつ、現状維持のリスクを淡々と伝える。

## 4-5. 異論別の受け止めフレーズ集

### A. 相手が反論してきた時（真正面で戦わない）

- 「なるほど、そこが一番引っかかってるんですね」
- 「めちゃくちゃ大事な観点です。その点を踏まえて進めていきましょう」
- 「そこが心配だから慎重に見たい、ってことですよね」
- 「一旦、○○さんにとっての納得できる軸を確認して進めさせてください」

### B. 条件が高すぎる時（否定せず現実へ）

**TIP（心理学）｜まず認めることで認知的不協和を回避**
「希望を持つのが自然」とまず肯定することで、候補者の自己肯定感を守る。否定から入ると防衛反応が働き、話が聞けなくなる。肯定→「一緒に現実も見よう」の順番で、候補者自身が現実を検討できる心理的余地をつくる。

> 「希望として持つのは自然です。むしろ最初に高い基準を置くのはとても大事です。」
「一方で実現可能性も、あわせて一緒に見ませんか。希望は希望、現実の勝ち筋をあわせて設計させていただくことで〇〇さんにとって後悔のない進め方ができればと思っています。」
「選択肢を3つに分けて一緒に設計してもいいですか？（挑戦/現実/安全）」
> 

### C. 施工管理が嫌だ（感情の受容 → 原因特定）

> 「施工管理ってしんどい経験をした人ほど嫌になりますよね」
「嫌な理由って、仕事内容そのものですか？それとも環境（人・残業・板挟み・責任）ですか？」
> 

→ 環境が原因なら、環境の良い施工管理求人で攻略可能。仕事内容そのものが原因なら、別キャリアパス（設備管理等）への誘導。

### D. 半年先で考えている（今動かない正当性の尊重）

> 「半年先で考えるの、むしろ正しいです。焦って動く方が事故ります。」
「その上で、半年後に動くのであれば今、キャリアの設計をして法人を見始めるのがちょうどいいです。他の方も大体半年後を見据えて動かれています。」
> 

## 4-6. 未経験者 vs 経験者：Phase 2での違い

| 対象 | 候補者の特徴 | CAの介入 |
| --- | --- | --- |
| **未経験者** | 情報の非対称性が非常に大きい。業界のことを知らない。ネット検索・YouTube・エージェントへの依存度が高い | 業界の全体像を丁寧に教育する。未経験から入った先輩の事例を複数紹介する。適性診断的なヒアリングで「向いている」フィードバックを与える |

# 第5章｜Phase 2.5：接点維持と法人打診（Phase 2 → 3の橋渡し）

## 5-1. 架電後の即時アクション

- **履歴書作成と同時に法人に打診をすぐに行う（即日）**
- 最初は一定条件がずれていても法人に電話で確認する
- 求職者の意向にあるものから順番に出していく
- ニーズが高いが返信が来ない場合は、逆オファーOKをもらえたところからどんどん出す
- 打診時は法人に書類選考＋電話で補足し、なるべく書類通過できるようにフォローする

## 5-2. JOB打診の優先順位管理

打診する会社の**優先順位を整理してだせる**状態にしておく。どの会社にどの順位をつけたか、**その理由（根拠）もわかるように**記録する。法人架電チームとの連携ルールは第12章 12-3を参照。

### 優先順位の整理単位

打診前に、候補企業を一覧化し、**優先順位（1位、2位…）** と **優先順位をつけた理由** をセットで管理する。

| 項目 | 内容 |
| --- | --- |
| 会社名 | 打診対象の法人名 |
| 優先順位 | 1位、2位、3位…（打診順と同じにすることが望ましい） |
| 優先順位の根拠 | なぜその順位にしたか（下記参照） |

### 優先順位をつける際の基準（例）

| 基準 | 説明・例 |
| --- | --- |
| 候補者のニーズ合致度 | 年収・エリア・勤務形態・キャリアパスなど、ヒアリング内容との一致度が高い |
| 逆オファー可能性 | 逆オファーOKをもらえている、または取りやすい |
| 応答速度・返信履歴 | 過去の打診で返信が早い、成約実績がある |
| 候補者の希望順位 | 候補者が「特に気になる」と表明した企業 |
| 入社希望時期との整合性 | 候補者が「即入社可能」の場合は、即戦力ニーズが高い法人を優先 |

※上記は一例。候補者ごとに重視する基準が異なる場合は、その理由を明記する。

### だし方（共有・出力の形）

- **Slack進捗共有時**：打診リストを「優先順位＋根拠」付きで投稿できるようにする
- **引き継ぎ時**：次の担当者が「なぜこの順番で打診したか」を理解できる状態にする
- **候補者説明時**：「〇〇さんのご希望に近い順にご紹介しています」と根拠付きで伝えられる

### 記載例

```
【打診優先順位】候補者：〇〇様

| 順位 | 法人名 | 優先理由 |
| 1 | A社 | 年収・エリア・休日数が希望に最も合致。逆オファーOK取得済み |
| 2 | B社 | キャリアパスが希望ルートと一致。応答が速い |
| 3 | C社 | 条件はやや控えめだが即入社希望に合致。候補者が興味表明 |
```

## 5-3. 連絡頻度の管理

**2日以上連絡がつかない状態は絶対に避ける。用がなくても上手く用事を作る。**

| 状況 | 対応 |
| --- | --- |
| 法人打診中で結果待ち | 「20社にアプローチしています。お待ちください」 |
| 法人からオファーが取れず間が空く | 「10社アプローチして、3社お断りされました。7社は検討いただいてます」 |
| 上記のお断り理由の伝え方 | **転職でネックになりそうな理由を先に伝えておく**（後でサプライズにならないように） |

**TIP（心理学）｜コミットメントと一貫性の原理**
人は一度公に選んだ行動を一貫して続けようとする傾向がある。打診優先順位を「根拠付きで共有する」ことで、CA自身も候補者も「なぜこの順番か」を自覚し、後の判断がぶれにくくなる。曖昧なまま進めると、途中で方針が揺らぐ。

## 5-4. コミュニケーションルール

- LINE電話/Zoom電話の後は、**必ず話した内容をLINEで送る**（内容のずれ防止）
- 連絡が繋がらない時の最終手段：「体調を崩しているのではないかと心配していました。大丈夫でしょうか？」

# 第6章｜Phase 3：評価・比較（Evaluation）

## 6-1. このフェーズの候補者心理

書類選考通過や面接を経て、具体的な企業・ポジションが候補に上がる段階。候補者の中で本格的な比較検討が始まる。

**候補者がやっていること：**

- 求人条件の詳細比較（年収、勤務地、仕事内容、福利厚生）
- 面接での企業理解の深化（社風、上司の人柄、現場の雰囲気）
- 現職との比較（残留した場合の将来 vs 転職した場合の将来）
- 家族・パートナーへの相談と合意形成
- 他社選考状況との並行比較

**構造的特徴：** 候補者の頭の中に「比較マトリクス」が形成される。しかしこのマトリクスは往々にして不完全であり、感情的なバイアスによって歪んでいる。CAの役割は、この比較マトリクスを**正確かつフェアに整理する**こと。

**TIP（心理学）｜選択のパラドックスと満足化**
選択肢が多すぎると満足度が下がる（選択のパラドックス）。「最高の選択」を追い求めすぎると決められなくなる。CAは「7つまでに絞る」「重み付けで優先順位をつける」など、選択肢を構造化し、候補者が「十分に良い」選択で満足できる（満足化）ように支援する。

## 6-2. 意思決定の6カテゴリ（候補者が考慮する要素）

候補者の頭の中を理解するために、転職判断に影響する要素を6カテゴリに分類する。CAはヒアリングを通じて、候補者がどのカテゴリを重視しているかを把握する。

### ① 経済的要素（Economic）

年収（手取りベースでの比較が重要）、昇給見込み（3年後・5年後）、福利厚生（社会保険、住宅手当、家族手当、退職金、資格手当）、固定給と変動給の比率、転職コスト（引越し、通勤費変化、退職金喪失）

### ② キャリア要素（Career）

スキル獲得の機会、資格取得支援（費用負担・期間）、キャリアパス（3年後・5年後・10年後のポジション）、市場価値への影響（この転職が次の転職にどう影響するか）、裁量権・意思決定への関与度

### ③ 労働環境要素（Work Environment）

通勤時間・転勤の有無・現場の場所、残業時間・休日出勤の頻度、年間休日数・有給取得率、シフト制か固定か・夜勤の有無・直行直帰の可否、労災リスク・安全管理体制（建設業では特に重要）

### ④ 組織・人間関係要素（Organization & People）

企業文化（体育会系かフラットか）、上司の人柄・マネジメントスタイル、チームの雰囲気・年齢層・技術レベル、会社の成長性・業績推移、経営者のビジョンと姿勢

### ⑤ 心理的要素（Psychological）

自己効力感（「自分にできるだろうか」）、変化への耐性（新しい環境への適応力）、帰属意識（現職への愛着、同僚との関係を断つことへの抵抗）、自尊心（「選ばれた」「必要とされている」感覚）、不確実性への態度

### ⑥ 社会的要素（Social）

家族の意向（特に転居を伴う場合は最大の変数）、世間体（転職回数、業界の社会的評価）、同業者の評判、タイミング（子供の学年、住宅ローン、親の健康状態）

## 6-3. CAのアクション：判断軸の可視化（Decision Matrix）

候補者の「なんとなくの比較」を、明示的な判断フレームワークに変換する。

**手法：**

1. 候補者に自分にとって重要な要素を挙げさせる（最大7つ）
2. 各要素に重み付けをさせる（合計100ポイント）
3. 各選択肢（現職 / A社 / B社）を10点満点で採点させる
4. 重み × 点数の合計で比較する

| 要素 | 重み | 現職 | A社 | B社 |
| --- | --- | --- | --- | --- |
| 年収 | 30 | 5（150） | 8（240） | 7（210） |
| 通勤時間 | 20 | 3（60） | 7（140） | 6（120） |
| スキルアップ | 25 | 4（100） | 7（175） | 8（200） |
| 人間関係 | 15 | 6（90） | ?（?） | ?（?） |
| 安定性 | 10 | 8（80） | 6（60） | 5（50） |
| **合計** | **100** | **480** | **615+** | **580+** |

**ポイント：数字そのものより、重み付けの過程で候補者自身が「自分が本当に何を重視しているか」に気づくことが最大の価値。**

**TIP（心理学）｜メタ認知の活性化**
重み付けや採点の作業は、本人に「自分の価値観を言葉にさせる」メタ認知を促す。無意識の判断を意識化することで、感情的なバイアスに気づき、より納得感のある意思決定ができる。CAは「なぜその重みにしたか？」と聞くことで、さらに深掘りできる。

## 6-4. CAのアクション：面接設定・面接準備

**面談実施2〜3日前に求職者と電話で接点を取りに行く。**

- **仮に内定が出た際の入社意向を確認する**（Phase 4への予防線）

**面接後の感想回収（面接後当日に回収し、Slackで伝達）：**

| 回収項目 | 目的 |
| --- | --- |
| 魅力に感じた点 | 候補者のPull要因の確認 |
| 面接で感じた懸念点 | 不安の先回り解消の材料 |
| **現状候補企業の中での希望順位** | 他社エージェント推薦企業も含め、その都度確認 |

## 6-4bis. 他社エージェントが介在する場合：他社エージェントを使わない要素

候補者が他社エージェントと並行して転職活動をしている場合、**他社エージェントに頼らず自社主導で進める**ことを原則とする。

### 判定要素（他社エージェント介在の有無）

以下が該当する場合、他社エージェントが介在していると判断する：

| 要素 | 具体例 |
| --- | --- |
| 書き起こし・ヒアリングで言及 | 「〇〇エージェントからも紹介されてます」「別の会社の担当者とも話してます」 |
| 候補企業の出所 | 他社エージェント推薦の企業が候補に含まれている |
| 選考状況の情報源 | 候補者の希望順位・進捗を他社エージェント経由で把握している |

### 使わないようにする要素（他社エージェント依存の排除）

| 避けるべきこと | 代わりにすべきこと |
| --- | --- |
| 他社エージェント推薦企業を優先して提案する | 自社開拓・自社推薦企業を中心に候補を組む |
| 他社エージェントの進捗・判断に合わせて動く | 自社のタイムラインと候補者の意思を軸にスケジュールを決める |
| 他社エージェント経由の情報を前提にFB・判断する | 自社面談・自社ヒアリングの内容のみを事実として扱う |
| 「他社と併用で」と候補者に促す | 「当社経由で進めるメリット」を伝え、一本化を検討してもらう |

### 基本スタンス

> 「他社エージェントと並行されている場合も、当社経由の求人・面接は当社主導で進めさせていただきます。他社の紹介状況に合わせるのではなく、〇〇さんにとっての最適な選択肢を、当社の情報をもとに一緒に整理していきましょう。」

## 6-5. 不安の先回り解消

候補者が言語化できていない不安を先回りして潰す。

### 未経験者への先回り

未経験者の最大の不安は**「自分にできるのか」**。これを解消するには抽象的な励ましではなく、具体的な情報が必要。

- 「入社後、最初の3ヶ月はこういう流れです」（具体的なオンボーディングプラン提示）
- 「同じように未経験で入った〇〇さんは、最初はこうでしたが、今はこうなっています」
- 「正直に言うと、最初の半年はきついです。でもそれを超えると△△になります」
- 「資格は□□を目指すことになります。平均的には○ヶ月で取得しています」

**未経験者の意思決定ハードルの構造：**

```
「興味はある」
    ↓
「でも自分にできるのか？」   ← 自己効力感の壁
    ↓
「年収が下がるのでは？」     ← 経済的リスクの壁
    ↓
「家族にどう説明する？」     ← 社会的承認の壁
    ↓
「今の仕事を辞めてまで？」   ← 損失回避の壁
    ↓
「この会社で大丈夫か？」     ← 情報不足の壁
    ↓
  意思決定
```

### 経験者への先回り

経験者の最大の不安は**「条件は本当に良くなるか」**と**「入ってみたら話と違った」**。

- 「この会社の中途入社者の割合は○%で、管理職の△%が中途です」
- 「年収テーブルはこうなっていて、〇年後にはこのレンジです」
- 「残業の実態は、月平均○時間。繁忙期は△時間です」
- 「現場は主に□□エリアで、直行直帰は可能です」

**経験者の意思決定ハードルの構造：**

```
「もっと良い条件があるはず」
    ↓
「この会社の条件は本当にベストか？」   ← 最適化の罠
    ↓
「面接で聞いた話は本当か？」          ← 信頼性の壁
    ↓
「今の現場を途中で抜けていいのか？」   ← 義理・責任感の壁
    ↓
「上司に退職を言えるか？」            ← 退職交渉の壁
    ↓
「引き止められたらどうする？」         ← カウンターオファーの壁
    ↓
  意思決定
```

# 第7章｜Phase 4：意思決定（Decision）

## 7-1. このフェーズの候補者心理

内定（オファー）を受け、承諾するかどうかの最終判断を行う段階。転職プロセスの中で**最も心理的負荷が高く、離脱リスクも最大**。

**意思決定を阻害する心理的障壁：**

| 障壁 | メカニズム |
| --- | --- |
| **現状維持バイアス** | 変化のリスクを過大評価し、現状のコストを過小評価する |
| **損失回避** | 「得るもの」より「失うもの」に注意が向く（同僚との関係、慣れた環境、退職金） |
| **選択のパラドックス** | 選択肢が多すぎると決められなくなる |
| **カウンターオファー** | 現職からの引き止め（昇給・昇格の提示）が判断を揺らがせる |
| **承認欲求** | 周囲（家族、友人、同僚）の反応を気にして決断が遅れる |
| **後悔回避** | 「あとで後悔するのではないか」という恐怖 |

**TIP（心理学）｜認知的不協和の軽減**
自分の行動と信念が矛盾すると不快感（認知的不協和）が生じる。カウンターオファーで揺れるのは「転職する」という決意と「現職が良くなるかも」という情報の矛盾。「根本的な理由はお金ではない」と気づかせることで、不協和を解消し、意思決定を後押しする。

## 7-2. CAのアクション：内定承諾面談

### カウンターオファー対策（事前に実施することが鍵）

内定承諾前に、現職からの引き止めシナリオを想定させ、対処方針を固めさせる。

> 「退職を伝えると、おそらく引き止められると思います。具体的には、昇給の提示や異動の提案があるかもしれません。その場合、どうされますか？」
> 

> 「カウンターオファーを受けて残った方の多くが、1年以内に再び転職活動を始めるというデータがあります。なぜなら、引き止めの条件は一時的な対処であり、あなたが転職を考えた根本的な理由は解消されないからです。」
> 

> 「もし仮に、今の会社が年収を50万上げると言ったら、残りますか？それでも残らないなら、あなたの転職の本当の理由はお金ではないということです。そこを確認しておきましょう。」
> 

### 家族への説明支援

候補者が家族を説得するための「武器」を渡す。一流のCAは候補者にこれらの情報を**整理した資料として**渡す。口頭の説明だけでは家族に正確に伝わらない。

| 家族の懸念 | 準備すべき情報 |
| --- | --- |
| 「収入は大丈夫？」 | 年収比較表（現職 vs 転職先、3年後の見込み含む） |
| 「安定してるの？」 | 会社の業績、業界の将来性、離職率のデータ |
| 「引越しは？」 | 勤務地の詳細、通勤シミュレーション |
| 「ブラックじゃない？」 | 残業実態、有給取得率、具体的な働き方 |
| 「なんで今？」 | 転職の動機を家族目線で翻訳した説明 |

### 未経験者 vs 経験者：Phase 4の違い

| 対象 | 離脱パターン | CAの介入 |
| --- | --- | --- |
| **未経験者** | 「決めきれない」（情報不足による不安で足踏み）。面接〜内定承諾で離脱しやすい | 意思決定の期限を設定。最大の不安を特定しピンポイントで解消する情報を提供。背中を押すタイミングを見極める |
| **経験者** | カウンターオファーに揺れる。内定承諾〜退職交渉で離脱しやすい | カウンターオファー対策を事前実施。家族への説明材料を提供。退職交渉のシナリオをリハーサル |

# 第8章｜Phase 5：実行・定着（Action）

## 8-1. このフェーズの候補者心理

意思決定後も、退職交渉での引き止め、入社前の不安（バイヤーズリモース＝買い手の後悔）、入社後のギャップなど、多くのリスクが存在する。

## 8-2. CAのアクション：退職交渉のコーチング

退職交渉のシナリオを事前にリハーサルさせる。

- 退職の切り出し方（誰に、いつ、どの場所で）
- 引き止めへの返答スクリプト
- 引き継ぎ計画の提示（「迷惑をかけない」姿勢を示す）
- 退職届の書き方・提出タイミング
- 法的権利の確認（退職は労働者の権利であること）

## 8-3. CAのアクション：入社前フォロー

入社前に発生する「バイヤーズリモース（本当にこれでよかったのか）」への対応。定期的な連絡で不安を解消し、入社日まで伴走する。

## 8-4. CAのアクション：入社後の定着支援

入社後のギャップを最小化し、早期離職を防ぐ。

| 時期 | フォロー内容 |
| --- | --- |
| 入社1週間後 | 「初日どうでしたか？」の確認。最初の印象をヒアリング |
| 入社1ヶ月後 | 仕事への慣れ、人間関係、想定とのギャップの確認 |
| 入社3ヶ月後 | 本格的な定着度チェック。不満・不安があれば企業側にもフィードバック |
| 入社6ヶ月後 | 試用期間後の評価確認。長期的なキャリアの方向性を再確認 |

# 第9章｜未経験者 vs 経験者 総合比較

全フェーズを通じた、未経験者と経験者の意思決定構造の違いを一覧する。

## 9-1. 全体比較表

| 比較軸 | 未経験者 | 経験者 |
| --- | --- | --- |
| **主な転職動機** | 業界チェンジ、キャリアリセット、将来性への投資 | 待遇改善、ステップアップ、環境改善 |
| **情報の非対称性** | **非常に大きい**（業界のことを知らない） | **比較的小さい**（業界を理解している） |
| **最大の不安** | 「自分にできるのか」「ついていけるか」 | 「条件は本当に良くなるか」「人間関係のリセット」 |
| **重視する要素の上位** | ①教育制度 ②将来性 ③職場の雰囲気 | ①年収 ②仕事内容 ③ポジション |
| **意思決定の速度** | 遅い（情報不足による不安で足踏み） | 比較的速い（判断基準が明確） |
| **離脱しやすいフェーズ** | Phase 3〜4（面接〜内定承諾） | Phase 4〜5（内定承諾〜退職交渉） |
| **カウンターオファーの影響** | 小さい（そもそも業界を変えたい） | **大きい**（現職が条件を上げてくる） |
| **家族の影響度** | 中〜大（「なぜわざわざ未知の業界に？」） | 中（「条件が良くなるなら」で納得されやすい） |

## 9-2. 未経験者に響く訴求ポイント

- 「未経験OK」だけでは不十分。**「未経験からこうなれる」という変化のストーリー**が必要
- 資格取得支援の具体性（費用負担、勉強時間の確保、合格率）
- 「手に職がつく」「一生食いっぱぐれない」という安全保障メッセージ
- 先輩社員の実体験（入社前の不安 → 入社後の実態 → 現在の状況）

## 9-3. 経験者に響く訴求ポイント

- 具体的な数字（年収、案件規模、従業員数、離職率）
- 「あなたの経験が活きる」という承認メッセージ
- 業界内での企業ポジション（競合との比較）
- 中途入社者の活躍事例と昇進実績
- 経営陣の方針と会社の方向性

# 第10章｜一流コンサルの6原則

## 原則1：候補者の意思決定プロセスの「現在地」を常に把握している

候補者がどのフェーズにいるかで、必要な支援がまったく異なる。

## 原則2：情報の非対称性を埋める役割を自覚している

未経験者にとってCAは「業界の翻訳者」。経験者にとっては「内部情報の橋渡し役」。候補者が自分で判断できるだけの十分な情報を、適切なタイミングで提供する。

## 原則3：「決めさせる」のではなく「決められる状態をつくる」

候補者を操作しない。材料と構造を提供し、候補者が「自分で決めた」と感じる状態をつくる。

## 原則4：短期の成約ではなく、長期の信頼を優先する

無理に転職させても早期離職になれば全員が損をする。「今回は転職しない方がいい」と言えるCAが、結果的に最も多くの成約を上げる。

## 原則5：感情と論理の両方を扱える

転職は論理的な判断に見えて、実は極めて感情的な意思決定。候補者の感情（不安、期待、恐怖、プライド）を正確に読み取りながら、論理的なフレームワークで整理する。

## 原則6：候補者の人生に対する敬意を持っている

転職は候補者の人生における重大な決断。数字や成約率の向こう側に一人の人間の人生があることを忘れない。この姿勢が候補者からの信頼と、結果としてのリファラルを生む。

# 第11章｜実例から学ぶケーススタディ（Slack実績ベース）

以下のケースはすべてXCHANGEの実際の対応記録（Slack / CA進捗チャンネル）から抽出し、マニュアルの理論フレームワークとの対応関係を注記したものである。個人情報は匿名化している。

## Case 1：前原さん ― 「独立したい経験者」の引き止め突破に失敗

**候補者プロフィール：** 電気工事経験者。現職で働きながら将来の独立を志向。鈴鹿（元請け案件多数の企業）への転職を提案。

**結果：最終的に辞退（Phase 4で離脱）**

### 経緯と時系列

**Phase 2-3（評価・比較）**

CAチーム（林・いけお）は、候補者の「独立したい」という深層動機に対して、鈴鹿転職が独立準備に最適であるという論理を構築した。

具体的に提示した比較軸：

- **資金面：** 現職のまま独立 vs 鈴鹿経由で独立 → 5年間で1,100万円超の年収差
- **コネクション：** 鈴鹿は元請け・ゼネコンとの直接取引が多く、独立後の受注先ネットワークを自然構築可能
- **資格・経験：** 施工管理技士取得支援あり。特高圧・太陽光・情報通信など幅広い案件で経験の幅が拡大

CAは独立成功に必要な「3大定量指標」まで踏み込んで設計した：

- 登録電気工事業者になるための自己資金250万円
- 発注者の知り合い50人、仕事をくれる会社5社
- 対応できる工事の幅の拡大

**Phase 4（意思決定）― ここで崩れた**

候補者は現職から引き止めを受け、「1億規模の施工管理を任せる」という話が出た。候補者の心理は以下のように揺れた：

> 「現職独立と鈴鹿独立で半々。現職の懸念は1億くらいの工事の施工管理を任せてもらうという話があり、そこを経験して独立が良いのではないかと。でも安定や仕事の量を考えると鈴鹿から独立した方が良さそうだし悩んでいる。」
> 

最終的に候補者は辞退。理由は：

> 「入ってすぐに辞めるのも迷惑がかかる。現職にも迷惑がかかる。迷惑をかけずに独立したい。」
> 

### 失敗要因の分析（マニュアル理論との対照）

**① カウンターオファー対策の遅れ（第7章 Phase 4）**

カウンターオファーの事前シナリオプランニングが不十分だった。「もし現職が大型案件を任せると言ったら？」という想定を内定承諾前に行うべきだった。

**② 候補者の懸念ベクトルの受け止め不足（第1章 原則②）**

チーム内FBで指摘された重要な反省：

> 「最初あんまり受け止めてない。相手が懸念抱えていそう。相手の不安は否定することなく、『そうなんですね』じゃなくて『重要なポイントですよね』と聞いていかないと、"こいつわかってくれない"となる」
> 

> 「基本的に1問1答になってしまっている。相手は独立をとにかくしたい。リスクだけ言っても、リスクはわかっているんだけどってなる。どうしたら独立を成功させるかで話を進めないと距離を置かれる」
> 

**③ 「迷惑をかけたくない」の受け止め方（第4章 異論別フレーズ集）**

候補者の「迷惑をかけたくない」を論理で否定しようとしたが、チーム内で以下の改善案が出た：

> 「"会社に迷惑がかかる"がそんなことないよの払拭方向でいくのがよかったかも。相手の意向のベクトルを受け止めて解釈を変えるのが大事。どうしてそう思っているなら、そこが違うなら事実を提供。短期間でも働いてくれることは法人にとってもありがたいと。」
> 

### このケースから追加すべきOPSルール

> **【追加ルール①】独立志向の候補者には「独立の成功支援者」としてポジションを取る**
転職を「独立への足かせ」ではなく「独立成功の最短ルート」として位置づける。そのためには独立後の事業計画レベルまで一緒に設計する姿勢が必要。1問1答のヒアリングではなく、候補者の理想の未来を膨らませる対話をする。
> 

> **【追加ルール②】カウンターオファー対策は「内定が出る前」に完了させる**
「もし現職がこう言ったら？」のシナリオを、できれば面接前の段階で候補者と話しておく。内定後では遅い。
> 

> **【追加ルール③】「迷惑」系の懸念は正面否定せず、事実で中和する**
「迷惑じゃないですよ」ではなく、「短期間でも貴重な戦力として歓迎されます。実際に法人側もそう言っています」と事実ベースで安心材料を提供する。
> 

## Case 2：鈴木さん ― 経験者の複数社同時進行を制御し内定獲得

**候補者プロフィール：** 2級電気工事施工管理技士・1級補を保有。1社経験で職務経歴書のクオリティが高い。神奈川在住、年収500万以上希望。

**結果：複数社から面接オファーを獲得し選考進行（Phase 3-4進行中）**

### 経緯のポイント

**Phase 2（探索・接触）：即日法人打診の実行**

候補者の登録後、CA（林）は即日で複数社に書類推薦を実施。鈴鹿・ザイマックス・幸信電気・ヤマト電機・正興電機・梅光園など同時に6社以上にアプローチ。

チーム内のやり取り：

> 「この履歴書、若いし転職歴ないし資格あるし、経験ささるところには良さそうに見える」
「500万いけるならどこでもいい系で、2級電気施工管理も持っている」
> 

**Phase 3（評価・比較）：突発事態への即時対応**

面接当日、候補者が現場作業中に熱中症で倒れるトラブルが発生。チーム内で即座にリスケ対応が動いた。

重要なFB：

> 「さすがに体調不良ではなくて熱中症まできたから、メールやなくて電話で対応してもらえると」
「メール入れる時は電話したことを記載するといいですよ」
> 

**面接への同席対応：** CAが面接にカメラOFFで同席し、候補者のコミュニケーション課題を把握。

> 「この人まじコミュ力きついんよな」 → だからこそCA側で面接を観察し、改善ポイントを直接フィードバックする体制を取った。
> 

### このケースの成功要因

> **【成功パターン①】資格保有経験者は即日法人打診 → 複数社同時進行で選考スピードを上げる**
履歴書の質が高い候補者は市場で「売れる」。CAが最初に動くスピードが成約率を決める。推薦文の品質チェックも重要（「ポジションが出すポジションと違ったり、記号が大量についている」等の指摘 → 修正してから送る）。
> 

> **【成功パターン②】突発事態（体調不良等）はメールではなく電話で即対応**
法人への印象管理が成約率に直結する。メールだけでは温度感が伝わらない。
> 

> **【成功パターン③】コミュ力に不安がある候補者には面接同席で補完する**
CA側がカメラOFFで面接に入り、リアルタイムで候補者の受け答えを把握。面接後に具体的なFBが可能になる。
> 

## Case 3：平野さん ― 未経験者の「どこでもいい」を面接突破に導く

**候補者プロフィール：** 電気工事士資格なし。イベント設営と電気工事の経験あり。糖尿病の持病あり。神奈川在住。

**結果：複数社面接設定に成功（テクノプロ書類通過、八興・ナカノエンジニアリング等に進行）**

### 経緯のポイント

**Phase 2-3（探索・評価）：資格なし候補者の法人攻略**

資格がない候補者を推薦する際、法人から以下のような質問が来た：

> 「電気屋さんとイベント設営が長く、電気の現場仕切りは問題ないかと思いますが、電気工事士の免許資格はお持ちではないのでしょうか。」
> 

CAチーム内でのやり取り：

> 「電気工事士の資格なくても行けるかだけ面談前に確認取れてると嬉しい」
> 

→ 法人に事前確認を入れることで、無駄な面接設定を防ぐ。

**健康面の配慮事項の先回り対応：**

候補者に糖尿病があり、深夜・早朝シフトは生活リズムが崩れて厳しい。CAチーム内で以下のように整理：

> 「深夜と早朝がない勤務は無理か確認できないかな？糖尿病があるから、できれば安定して働きたい」
「生活リズムが合っていればOKで、そうじゃないときつそう。SECOMとかはそういう意味ではOKだったけど落ちちゃった」
> 

→ 候補者の健康事情を理解した上で、勤務体系が合う企業を優先的にマッチングする。

**「8月入社」のスピード感：**

> 「平野さん8月入社できるのが熱い。変な話7月からでもすぐに働きたい」
> 

→ 入社希望時期の早さは法人にとっても魅力。これを逆オファーの交渉材料に使う。

### このケースから追加すべきOPSルール

> **【追加ルール④】資格なし候補者は推薦前に法人に「資格なしでも選考可能か」を確認する**
書類を送ってから「資格必須」で落ちるのは時間のムダ。事前に電話で確認することで、候補者の心理的ダメージも防ぐ。
> 

> **【追加ルール⑤】健康上の配慮事項は、法人への伝え方を候補者と事前に合意する**
持病がある候補者の場合、「勤務体系の希望」として伝えるか、「健康上の理由」として開示するかを候補者と事前にすり合わせる。
> 

> **【追加ルール⑥】入社希望時期が早い候補者は交渉カードとして活用する**
法人にとって「すぐに来てくれる」は大きな魅力。「即入社可能です」を逆オファーの際に前面に出す。
> 

## Case 4：組織の歪みチャンネルから学ぶ ― 繰り返し発生するオペレーション課題

XCHANGEでは「歪み（yugami）チャンネル」に構造的な課題が蓄積されている。以下は実際に議論された代表的な歪みとその解決策である。

### 歪み①：求職者のニーズ深掘りが甘く、的外れな求人を紹介して熱が冷める

**発生した問題：**

> 「求職者のニーズの深掘りまでできるように初手対応したらすぐにXCHANGEにパスするオペレーション構築してほしい」
> 

> 「求職者のニーズ・優先度をしっかり把握できていない → 的はずれな求人紹介して熱が冷めていく問題」
> 

> 「優先項目として『年収400万以上』『休みが多い』があった時に、『休み少ないけど400万以上ならいい』のか『400万切っちゃうけど休みは超絶しっかりしてる』ならいいのか、この微妙なラインを把握できていない」
> 

**チームで議論された解決策：**

- ニーズの優先順位を管理ツール（Trello/Zoho）に明記する
- ただし管理項目を増やしすぎると生産性が落ちるため、メモ欄に自由記載で運用

**マニュアルへの反映 → 第4章 ヒアリング③の「ワークライフバランス vs 年収の比重確認」が、まさにこの歪みを解消するために設計されている。**

### 歪み②：法人獲得後の求人情報が社内共有されない

**発生した問題：**

> 「法人獲得後の求人共有ができていない。法人獲得のたびに、求人の共有 → 既存求職者で該当いないかチェック、するフローを組みたい」
> 

**解決策：** 法人紹介OK取得 → 即時Slackで求人概要を共有 → 既存候補者リストとのマッチング確認、を標準フローに組み込む。

### 歪み③：妥当年収がわからずヒアリング時に修正できない

**発生した問題：**

> 「妥当年収がそもそも分からないので、ヒアリング時に適正年収に修正できない」
「職種・免許・もしあれば夜勤有無などの条件でだいたいの平均があると話しやすい」
> 

**マニュアルへの反映 → 第4章 ヒアリング④のキャリアパス別年収レンジ表がこの歪みへの回答。**

### 歪み④：未通電リストが無限に溜まる

**発生した問題：**

> 「未通電リストの期限を決めないとずっとあってだるいことになる」
「求職者への2日以内に通電をKPIにしてたっけ？」
> 

**マニュアルへの反映 → 第4章 4-2のリード管理オペレーション。未通電は2週間管理、3回以上架電してもつながらない場合はナーチャリングステージへ移行。**

### 歪み⑤：施工管理求人の実態が作業員採用である問題

**発生した問題（法人架電チームが発見）：**

> 「施工管理として求人出しているが、業務内容は作業員としての採用をしている」
「施工管理が現場作業してはいけない → 採用企業の違法行為になる」
> 

**対応方針：** 「言わないほうがいいですよ」のスタンスで法人に指摘。我々はあくまでも法律に則って紹介する。施工管理求人であれば施工管理として紹介可能。実際の職務内容に関しては関与できない。

**マニュアルへの反映 → 法人開拓時に「実際の業務内容」を確認するヒアリング項目として標準化する。**

# 第14章｜法人架電のゴールデンタイムと求職者対応の時間設計

## 14-1. 法人架電のゴールデンタイム

| 時間帯 | 接続率 | 用途 |
| --- | --- | --- |

| 16:00-17:00 | **高** | 第2のゴールデンタイム |

```
【平日の時間設計】

  9:00-9:30   ：朝準備（前日の引き継ぎ確認、リスト選定、リマインドLINE送信）
```

この時間設計は鉄則ではなく、候補者の面接対策や緊急対応が入れば柔軟に調整する。ただし**「法人架電の時間を最大化する」**という原則は守る。
